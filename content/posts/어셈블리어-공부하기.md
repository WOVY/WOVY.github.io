## 어셈블리어란?

간단하게 말해 컴퓨터의 기계어와 1:1 대응되는 Low-level 언어입니다. CPU가 이해하는 ISA의 종류에 따라 기계어가 다르므로 어셈블리어 역시 아키텍처별로 다양하게 존재합니다. 드림핵 강의에서는 x64 아키텍처를 대상으로 하기에 x64 어셈블리어에 대해서 다루어 보겠습니다. 

또한 동일한 아키텍처라도 이를 표기하는 방식에 따라 대표적으로 Intel 문법과 AT&T 문법이 나뉘는데 이 글에서는 Intel 문법을 사용하는 X64 아키텍처를 기준으로 하겠습니다.

## 기본구조

x64 어셈블리어는 굉장히 간단한 문법을 지닙니다. 동사에 해당하는 **명령어(Opcode)** 와 목적어에 해당하는 **피연산자(Operand)** 로 구성됩니다.

## 명령어

x64에는 매우 많은 명령어가 존재하는데 이 중 아래의 21개의 명령어만 다루어 보겠습니다.

| 분류 | 명령 코드 |
| --- | --- |
| 데이터 이동(Data Transfer) | `mov`, `lea` |
| 산술 연산(Arithmetic) | `inc`, `dec`, `add`, `sub` |
| 논리 연산(Logical) | `and`, `or`, `xor`, `not` |
| 비교(Comparison) | `cmp`, `test` |
| 분기(Branch) | `jmp`, `je`, `jg` |
| 스택(Stack) | `push`, `pop` |
| 프로시저(Procedure) | `call`, `ret`, `leave` |
| 시스템 콜(System call) | `syscall` |

## 피연산자

피연산자에는 상수, 레지스터, 메모리의 총 3가지 종류가 있습니다. 이 중 메모리 피연산자는 `[]`으로 둘러싸인 것으로 표현되며 앞에 크기지정자인 `TYPE PTR`이 올 수 있는데, TYPE의 위치에는  BYTE, WORD, DWORD, QWORD가 올 수 있어 각각 1, 2, 4, 8바이트의 크기를 지정합니다.

## 산술 연산과 논리 연산

### add, sub, mul, div

#### 1. add

```
add <destination>, <source>
```

`<destinaion>`에 `<source>`를더한 결과를 `<destinaion>`에 대입한다 라고 해석하면 됩니다.

#### 2. sub

```
sub <destination>, <source>
```

덧셈과 비슷하게 `<destinaion>` 에 `<source>`를 뺀 결과를 `<destinaion>`에 대입합니다.

#### 3. mul / imul

mul, imul 명령어는 EAX 레지스터에 보관된 값과 피연산자를 곱하는 명령어입니다. 연산결과의 크기와 피연산자의 크기가 다르기 때문에 add, sub 명령어와 비교했을 때 처리가 복잡합니다. 

##### mul

```
mul <source>
```

mul 명령어는 암시적으로 RAX 레지스터를 첫 번째 피연산자로 사용하고 `<source>`를 두 번째 피연산자로 사용합니다. 앞서 말했듯이 피연산자의 크기에 따라 보관되는 레지스터가 바뀝니다.

- 8비트 x 8비트
AL 레지스터와 `source`의 값을 곱합니다. 연산 결과가 최대 16비트이기 때문에 AX(16비트) 레지스터에 보관합니다.
- 16비트 x 16비트
AX 레지스터와 `source`의 값을 곱합니다. 연산 결과의 상위 16비트는 DX 레지스터, 하위 16비트는 AX 레지스터에 보관합니다.
- 32비트 x 32비트
EAX 레지스터와 `source`의 값을 곱합니다. 연산 결과의 상위 32비트는 EDX, 하위 32비트는 EAX에 보관합니다.
- 64비트 x 64비트
RAX 레지스터와 `source`의 값을 곱합니다. 연산 결과의 상위 64비트는 RDX, 하위 64비트는 RAX에 보관합니다.

##### imul

부호가 있는 정수에 대해 곱셈을 수행하는 명령어입니다. 

```
imul <source> ; 피연산자가 1개
imul <destination>, <source> ; 피연산자가 2개
imul <destination>, <source>, <immediate> ; 피연산자가 3개
```

- 피연산자 1개 : mul과 유사하게 정해진 레지스터 (AL/AX/EAX/RAX)와 `<source>`를 곱한 결과를 AX/DX:AX/EDX:EAX/RDX:RAX에 보관합니다.
- 피연산자가 2개 : `<destination>에 <source>를 곱한 결과를 <destination>에 대입합니다.
- 피연산자가 3개 : `<source>`와 `<immediate>`을 곱한 결과를 <destination>에 대입합니다.

만약 연산 결과가 목적지 피연산자의 크기를 초과하여 데이터 손실이 발생하면 캐리 플래그 (CF)와 오버플로우 프래그 (OF)를 1로 설정하여 이를 알립니다.

#### div / idiv
div / idiv 명령어는 두 연산자에 나눗셈을 수행하는 명령어입니다. div는 부호가 없는 나눗셈을, idiv는 부호가 있는 나눗셈을 수행할 때 사용합니다.

```
div/idiv <제수>
```

피제수를 크기에 따라 AX, DX:AX, EDX:EAX, RDX:RAX에 저장하고, 수생한 결과인 몫과 나머지를 각 크기에 따라 AL:AH, AX:DX, RAX: RDX에 저장합니다.

### and, or ,xor, not
```
AND/OR/XOR <destination>, <source>
NOT <destination>
```
